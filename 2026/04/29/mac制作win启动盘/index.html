mac 制作 win 启动盘

这两天给家里的老 Windows 重装系统；重装的过程可以说是各种问题不断，捣鼓了好一阵儿～

自信满满

一开始拿自己曾经装系统的U盘直接对这台电脑进行系统重装。U盘可以被识别，但安装过程中却报了许多异常，疑似安装程序损坏，但这个过程前面那步已经将之前的系统全删了，现在只能通过U盘重装了。然而现在，需要重新制作一个U盘启动盘。

心烦气躁

因为目前手中只有一台可以正常使用的Mac和一台待装系统的Windows；所以这次制作终极U盘只能通过Mac制作Windows系统的装机U盘；

这个过程不像直接使用Windows那么简单，直接使用Windows可以直接对官网保存为U盘介质；

但本次使用Mac的话，就只能先下载ISO，然后重新格式化U盘，再将ISO具体内容移到U盘中；

当然在制作这个U盘的过程也是非常的坎坷的；

最一开始尝试的是直接使用第三方软件进行烧录；但它此时会将U盘生成如图这种；这种在Windows上重装的时候，并无法直接识别出U盘的内容；

而后又尝试了将这些文件直接复制到格式好的格式化好的U盘上；

但现在关于Windows系统的镜像文件也是越来越大的，FAT格式的U盘已无法承受将其移入卷宗；

其中的两个文件内容过大，导致之前格式化的格式不能够支持文件移入U盘；

为保证文件可以顺利地移入U盘，便尝试了另一种U盘的格式 ExFAT；

但通过这种格式制成一个Windows系统安装盘时，Windows电脑并不能准确地识别出这个U盘；

可能是因为这是一台老Windows，也可能是因为别的原因……

柳暗花明

而后又将U盘格式化为那个格式；

想办法将大文件移入这个U盘；

接下来就尝试了文件切割的方式 当然也是最终制作安装盘成功的方式；也发现了挂载其实就是烧录；

具体流程可以参考视频：

【【折腾必懂S2·第四期】Mac制作Win11启动盘并去掉Win联机安装-哔哩哔哩】 https://b23.tv/WpW6ez5

homebrew安装wimlib命令行工具

brew install wimlib

格式u盘

终端输入命令

rsync -avh --progress --exclude=sources/install.wim /Volumes/CCCOMA_X64FRE_ZH-CN_DV9/ /Volumes/WIN10

wimlib-imagex split /Volumes/CCCOMA_X64FRE_ZH-CN_DV9/sources/install.wim /Volumes/WIN10/sources/install.swm 4000

即实现了通过分割的方式将大文件移入FAT格式的U盘！

新的挑战

在完成新的装机U盘后，本以为接下来可以一切顺利，但却在读取安装文件并安装程序后重启时没有默认从U盘打开，导致又出现联网PXE问题；

这时需要在按开机键后疯狂按f2键进入BIOS将Boot List Option启动列表模式将legacy改为UEFI；

然后设置Boot Option#1改为U盘，按f10保存并退出；

这样再按f12进入BIOS设置界面时重启就会自动从U盘安装，不会再掉到PXE了！
