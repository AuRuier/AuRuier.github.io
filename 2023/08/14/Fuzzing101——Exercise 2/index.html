Fuzzing101——Exercise 2

This time we will fuzz libexif EXIF parsing library. The goal is to find a crash/PoC for CVE-2009-3895 （一种基于堆的缓冲区溢出，可通过无效的 EXIF 图像触发）and another crash for CVE-2012-2836 （一个越界读取漏洞，可通过带有精心制作的 EXIF 标记的图像触发）in libexif 0.6.14.

CVE-2009-3895 是一种发生在堆数据区的缓冲区溢出，通常与显式动态内存管理（使用 malloc() 和 free() 函数进行分配/去分配）有关。更多信息。

CVE-2012-2836 是指当程序读取的数据超过预定缓冲区的末尾或开头时发生的漏洞。更多信息。

学会AFL +的基本使用，学会复现crash。

目标

    1. 找到一个使用 libexif 库的接口应用程序
    1. 创建一个 exif 样本的种子语料库
    1. 使用 afl-clang-lto 编译 libexif 和所选应用程序
    1. 对 libexif 进行模糊测试，直到出现一些独特的崩溃
    1. 对崩溃进行分级，为每个漏洞找到 PoC
    1. 修复问题

下载安装目标

cd $HOME
mkdir fuzzing_libexif && cd fuzzing_libexif/

下载 libexif-0.6.14 并解压：

wget https://github.com/libexif/libexif/archive/refs/tags/libexif-0_6_14-release.tar.gz
tar -xzvf libexif-0_6_14-release.tar.gz

Build and install libexif:

cd libexif-libexif-0_6_14-release/
sudo apt-get install autopoint libtool gettext libpopt-dev
autoreconf -fvi
./configure --enable-shared=no --prefix="$HOME/fuzzing_libexif/install/"
make
make install

选择接口应用

此次采用 exif command-line. 下载并解压 exif command-line 0.6.15:

cd $HOME/fuzzing_libexif
wget https://github.com/libexif/exif/archive/refs/tags/exif-0_6_15-release.tar.gz
tar -xzvf exif-0_6_15-release.tar.gz

build and install exif command-line utility:

cd exif-exif-0_6_15-release/
autoreconf -fvi
./configure --enable-shared=no --prefix="$HOME/fuzzing_libexif/install/" PKG_CONFIG_PATH=$HOME/fuzzing_libexif/install/lib/pkgconfig
make
make install

使用fuzz编译器编译（afl-clang-lto）

rm -r $HOME/fuzzing_libexif/install
cd $HOME/fuzzing_libexif/libexif-libexif-0_6_14-release/
make clean
export LLVM_CONFIG="llvm-config-11"
CC=afl-clang-lto ./configure --enable-shared=no --prefix="$HOME/fuzzing_libexif/install/"
make
make install

cd $HOME/fuzzing_libexif/exif-exif-0_6_15-release
make clean
export LLVM_CONFIG="llvm-config-11"
CC=afl-clang-lto ./configure --enable-shared=no --prefix="$HOME/fuzzing_libexif/install/" PKG_CONFIG_PATH=$HOME/fuzzing_libexif/install/lib/pkgconfig
make
make install

tips:

这里采用了 afl-clang-lto 来替代 afl-clang-fast.
通常， afl-clang-lto 是最好的选择，因为它是 collision-free instrumentation ，比 afl-clang-fast 更快。

如果不确定何时使用 afl-clang-lto 或 afl-clang-fast ，那么可以查看下图 AFLplusplus : instrumenting that target

Fuzzing

AFL_I_DONT_CARE_ABOUT_MISSING_CRASHES=1 afl-fuzz -i $HOME/fuzzing_libexif/exif-samples-master/jpg/ -o $HOME/fuzzing_libexif/out/ -s 123 -- $HOME/fuzzing_libexif/install/bin/exif @@

Eclipse 设置

  • 练习 1 : 在上次练习中，学会了如何使用 GDB 控制台来分流崩溃。
  • 练习 2 : 此次练习将学习如何使用 Eclipse-CDT 进行调试。

Eclipse 下载：

https://www.eclipse.org/downloads/download.php?file=/technology/epp/downloads/release/2021-03/R/eclipse-cpp-2021-03-R-linux-gtk-x86_64.tar.gz

JAVA-SDK 下载：

sudo apt install default-jdk

解压：

tar -xzvf eclipse-cpp-2021-03-R-linux-gtk-x86_64.tar.gz

启动 Eclipse-CDT 后，将源代码导入 Project Explorer。然后，进入 File（文件）-> Import（导入）-> 然后选择 C/C++ -> “Existing code as makefile project”。然后选择 “Linux GCC “并选择 Exif 源代码文件夹。如下图：

配置调试参数。 Run -> Debug Configurations，然后选择 exif 项目并浏览 exif 二进制文件：

然后设置输入参数。进入"Arguments" 选项卡，设置 AFL 崩溃的路径：

最后，点击 "Debug" 开始调试，程序就会在主函数开始时停止。点击 Run -> Resume ，程序就会在检测到分段故障时停止执行。
